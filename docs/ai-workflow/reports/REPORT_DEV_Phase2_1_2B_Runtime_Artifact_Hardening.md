Apply target resolution was too dependent on source adjacency for some explicit clarification targets. Real artifacts showed tip-origin jobs with explicit `clarification-item` metadata but later application into a new `...normal-note` clarification, which matches the user-observed copied-tip failure. `findLiveClarificationMatch` now trusts an explicit live clarification id only when the job was truly sourced from a Learning OS clarification item or live inline draft target, while preserving the existing normal-note mismatch protection.

Root cause D:
Story requests from Learning OS items depend on generated fallback behavior when the merge proposal is partial or malformed. Runtime fixtures now preserve the real generated-content-item shape and verify that a usable AI story produces non-empty generated-content preview/draft. This shares part of Root cause A but is covered separately for generated-block and tip-source paths.
